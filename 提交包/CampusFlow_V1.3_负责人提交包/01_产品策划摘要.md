# CampusFlow 产品策划摘要

## 一句话定位

CampusFlow 是面向高校教务、学工、团委和场地管理部门的校园空间预约与审批智能调度助手，帮助学校把“学生找空间、社团借场地、老师审申请、管理员看运营”升级为可解释、可追踪、可验收的 AI 工作流。

## 核心问题

高校空间资源使用中存在四类高频问题：

| 问题 | 当前表现 | CampusFlow 的解决方式 |
| --- | --- | --- |
| 找空间难 | 学生靠经验、群聊或现场寻找 | 自然语言找空间，按时间、人数、设备、位置推荐 |
| 借场地慢 | 社团材料反复补充，审批前核对成本高 | 自动生成申请草稿，提示风险和材料清单 |
| 审批重复 | 老师要人工查课表、容量、设备、规则 | AI 预审 + 人工确认，降低重复核对 |
| 管理粗放 | 空间冲突、利用率和审批瓶颈难量化 | 管理员看板、周报摘要和试点验收指标 |

## 产品边界

CampusFlow 不是校园通用聊天机器人，也不替代学校已有教务、OA、统一身份认证或预约系统。它定位为现有系统之上的 AI 调度层和流程助手。

首版聚焦两条可验收闭环：

1. 学生或社团自然语言找空间。
2. 社团活动申请预审与人工审批流转。

明确不做：

- 不做校园万能入口。
- 不让 AI 绕过人工审批直接批准中高风险活动。
- 不使用个人门禁轨迹、成绩、处分、心理等高风险数据。
- 不在开发阶段导入客户真实隐私数据。

## 目标用户

| 角色 | 核心诉求 | 版本覆盖 |
| --- | --- | --- |
| 学生 | 快速找到可用空间 | 已覆盖 |
| 社团负责人 | 快速匹配活动场地，减少材料打回 | 已覆盖 |
| 老师 | 集中查看待审批申请、风险项和材料状态 | 已覆盖 |
| 管理员 | 查看运营指标、冲突、试点交付状态 | 已覆盖 |
| 信息办 | 关注权限、数据边界和审计 | 方案与研发治理已覆盖 |
| 管理者 | 关注试点价值和可量化 ROI | 方案与验收指标已覆盖 |

## 试点建议

首个试点建议从学工处或团委的社团活动场地申请切入，范围控制在一个学院、一组楼宇或一个活动中心，周期 4-6 周。

核心验收指标：

| 指标 | 目标 |
| --- | --- |
| 申请单字段完整率 | 不低于 85% |
| 风险识别命中率 | 不低于 80% |
| 场地推荐人工抽检通过率 | 不低于 90% |
| 人工审批保留率 | 100% |
| 审计日志完整率 | 100% |
| 平均审批处理时长 | 降低 30% 以上 |
| 管理员人工核对时间 | 降低 40% 以上 |

## 提交结论

CampusFlow 已从概念方案推进到 V1.3 试点交付版，具备向负责人汇报的完整材料和可运行 Demo。下一步建议选择一个受控试点范围，由业务负责人、信息办和审批老师共同确认数据授权、角色范围、验收指标和人工兜底流程。
