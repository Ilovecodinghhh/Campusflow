# CampusFlow Demo 交付说明

## 交付内容

本 Demo 已完成一个可直接打开的静态网页，用于演示 CampusFlow 智校空间调度助手。

入口文件：

```text
demo/index.html
```

核心文件：

| 文件 | 说明 |
| --- | --- |
| `demo/index.html` | Demo 页面结构 |
| `demo/styles.css` | 页面视觉样式 |
| `demo/app.js` | 可直接打开的页面交互、Mock 数据和渲染逻辑 |
| `demo/src/engine.js` | 可测试的推荐、解析、风险预审和运营摘要逻辑 |
| `demo/tests/engine.test.mjs` | 业务逻辑测试 |
| `demo/README.md` | 打开方式和文件说明 |

## 已实现链路

### 答辩演示模式

顶部新增 `3 分钟演示` 和 `10 分钟演示` 两种模式：

1. `下一步` 会按答辩顺序切换角色和默认输入。
2. 步骤条会标记当前讲述环节。
3. `演示视图` 会收起左右侧栏，突出中间结果区，适合投屏。

### 链路 1：自然语言找空间

示例输入：

```text
今晚 7 点后，帮我找一个离东门近、安静、有插座、适合 4 人讨论的地方。
```

页面会展示：

1. AI 工作流卡片。
2. 推荐空间卡片。
3. 推荐分、容量、设备、位置、占用率。
4. 推荐理由。
5. 不可用空间及原因示例。
6. 数据来源和安全边界。

### 链路 2：社团活动申请预审

示例输入：

```text
周五晚上 7 点，办 80 人 AI 分享会，需要投影和麦克风，可能有 5 个外校嘉宾。
```

页面会展示：

1. 主推活动场地。
2. 自动生成申请字段。
3. 风险等级。
4. 人数、外校嘉宾、设备、报告厅等风险项。
5. 审批人建议。
6. 不可用场地说明。
7. 数据来源和高风险人工确认边界。

### 链路 3：老师审批工作台

页面会展示：

1. 待审批申请队列。
2. AI 预审摘要。
3. 风险项 chip 点击提示。
4. 通过、退回、调场操作。
5. 右侧指标实时联动。

### 链路 4：管理员运营看板

示例输入：

```text
本周哪些空间冲突最多？为什么？给我一份下周优化建议。
```

页面会展示：

1. 本周空间冲突排行。
2. 周报摘要。
3. 下周优化动作。
4. 右侧固定运营指标看板。
5. 数据来源和安全边界。

## 演示建议

建议按以下顺序演示：

1. 打开 `demo/index.html`，选择 `3 分钟演示`。
2. 点击 `下一步`，依次展示学生、社团负责人、老师、管理员视图。
3. 在学生视图点击 `预约`，说明推荐采纳率联动。
4. 在老师视图点击风险项和 `通过/退回/调场`，说明人工兜底和指标变化。
5. 点击 `演示视图`，展示投屏版结果区。
6. 最后强调产品边界：AI 只做推荐、预审和解释，高风险审批仍由人工确认。
