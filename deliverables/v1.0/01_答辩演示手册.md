# CampusFlow V1.0 答辩演示手册

## 演示目标

答辩时不要把 CampusFlow 讲成“一个聊天机器人”。核心表达应当是：

> CampusFlow 是把自然语言理解、结构化数据查询、规则过滤、人工审批和运营复盘串起来的校园空间调度工作流。

演示要证明四件事：

1. 学生可以更快找到合适空间。
2. 社团可以减少活动申请返工。
3. 老师可以看到风险项和依据后再审批。
4. 管理员可以看到空间冲突和运营指标。

## 演示前准备

1. 打开终端，进入项目根目录。
2. 启动本机服务：

```bash
PYTHONPATH=apps/api python -m campusflow.server
```

3. 打开浏览器：

```text
http://127.0.0.1:8765
```

4. 点击页面右上方“重置演示”，让数据回到干净状态。
5. 保持浏览器缩放在 90%-100%，避免投影时右侧看板溢出。

## 3 分钟答辩脚本

### 0:00-0:25 痛点切入

“高校里找空教室、借活动场地、核对课表设备和审批材料，是一件高频但很分散的事。学生不知道哪里能用，社团反复补材料，老师要手动核对冲突，管理员很难看到空间利用率。”

### 0:25-0:50 产品定位

“CampusFlow 不是校园版 ChatGPT，而是校园空间预约与审批的 AI 工作流。AI 负责理解自然语言和解释依据，真正的权限校验、数据查询、规则过滤、审批写入和日志审计由确定性系统完成。”

### 0:50-2:10 Demo 主链路

1. 选择“学生”，点击执行。
   - 讲：“学生一句话表达时间、人数、位置和设备，系统返回推荐空间，并解释为什么可用、为什么排除其他空间。”
2. 选择“社团负责人”，点击执行。
   - 讲：“社团活动会自动生成申请草稿，系统识别人数字段、外校嘉宾、设备要求，并给出风险等级和审批人。”
3. 点击“提交申请”。
   - 讲：“AI 不直接批准，而是把结构化申请和预审依据交给人工审批。”
4. 选择“老师”，点击“通过”或“补材料”。
   - 讲：“老师看到风险项、审批人和材料要求后做最终决定。”
5. 选择“管理员”。
   - 讲：“管理员可以看到冲突空间 Top5、运营指标、周报摘要和审计日志。”

### 2:10-2:45 试点价值

“V1.0 的验收指标不是主观体验，而是找空间耗时、推荐采纳率、申请一次通过率、审批处理时长、冲突率和空间利用率。适合先在一个学院或一组楼宇试点。”

### 2:45-3:00 收束

“所以这个项目的价值，不是让 AI 直接替学校做决定，而是把校园空间管理变成可解释、可追踪、可验收的智能工作流。”

## 关键页面讲解词

### 学生视图

“这里展示的是自然语言找空间。系统不是只给一个答案，而是展示候选空间、评分、推荐理由和不可用原因，方便用户相信结果。”

### 社团负责人视图

“这里是活动申请预审。80 人、外校嘉宾、麦克风这些信息会触发审批规则，系统生成材料清单和审批人，但不会绕过人工审批。”

### 老师视图

“老师看到的不是原始聊天记录，而是结构化申请、风险等级、审批人和操作按钮。这是把 AI 输出变成可审批工作台。”

### 管理员视图

“管理员看到的是运营层结果，比如冲突空间、利用率、推荐采纳率和审计日志。这让试点能用指标验收。”

## 高频问题短答

**问：为什么不用现有预约系统？**  
答：CampusFlow 不替代现有系统，而是作为 AI 调度层，连接自然语言入口、规则预审和人工审批，降低查询和核对成本。

**问：AI 会不会乱订教室？**  
答：不会。V1.0 中 AI 只做理解、推荐和解释，预约写入、审批通过、权限校验都由确定性系统和人工确认控制。

**问：数据从哪里来？**  
答：试点需要空间基础表、课表、预约记录、设备状态、审批规则和审计日志。V1.0 Demo 使用 SQLite Mock 数据模拟这些表。

**问：如何衡量效果？**  
答：用找空间耗时、申请一次通过率、审批处理时长、冲突率、推荐采纳率和空间利用率做基线对比。

**问：如果接入真实学校系统，难点在哪里？**  
答：主要在数据治理和权限边界，不在前端页面。需要先明确空间、课表、预约、设备和审批规则的权威数据源，再做 API 对接。

## 答辩时避免的表述

- 避免说“AI 自动审批活动”。
- 避免说“已经能直接上线全校使用”。
- 避免说“可以接入所有校园数据”。
- 避免把 Demo 说成最终生产系统。

## 推荐结尾

“CampusFlow V1.0 已经具备方案、PRD、Demo、测试、试点验收和工程说明。下一步不是盲目扩功能，而是在一个学院或团委社团活动审批场景中试点，用真实数据和指标验证价值。”
