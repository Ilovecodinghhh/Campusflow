## 2. Demo 检查

提交或答辩前按顺序操作：

1. 启动后端：

```bash
PYTHONPATH=apps/api python -m campusflow.server
```

2. 打开：

```text
http://127.0.0.1:8765
```

3. 点击“重置演示”。
4. 执行学生视图，确认出现空间推荐。
5. 切换社团负责人，确认出现申请草稿。
6. 点击提交申请，确认出现申请编号。
7. 切换老师，确认能看到待审批申请。
8. 点击通过、补材料或改期，确认状态变化。
9. 切换管理员，确认运营指标、冲突 Top5、周报摘要、审计日志可见。

## 4. 答辩口径检查

| 问题 | 推荐回答方向 |
| --- | --- |
| 这是聊天机器人吗？ | 不是，是 AI 工作流和调度层 |
| AI 会自动审批吗？ | 不会，中高风险必须人工确认 |
| 数据安全吗？ | V1.0 不接个人轨迹，关键动作有审计 |
| 如何试点？ | 先从一个学院/团委社团审批场景开始 |
| 怎么证明价值？ | 用找空间耗时、一次通过率、审批时长、冲突率等指标 |
| 研发能落地吗？ | 已有本机 API、SQLite、前后端联调和测试 |

## 5. 风险提示

提交时建议主动说明：

1. V1.0 是本机可运行 MVP 和试点准备包，不是全校生产系统。
2. 当前自然语言解析为规则模拟，生产版可替换为 LLM/RAG。
3. 当前数据是 Mock/Seed 数据，试点需要学校提供权威数据源。
4. 当前权限为 Demo 角色映射，生产版需要统一身份认证。

## 6. 推荐提交摘要

> CampusFlow V1.0 已完成从项目方案到本机可运行 MVP 的闭环，并补齐答辩、试点、工程和提交检查材料。当前版本能演示学生找空间、社团申请、老师审批、管理员复盘，并通过自动化测试验证主要 API 与权限边界。建议作为提案提交，并在答辩后进入一个学院或团委社团活动审批场景的小范围试点。
